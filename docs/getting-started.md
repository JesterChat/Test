## Что такое GitHub?

GitHub - это облачное хранилище для твоего кода. Туда ты можешь:
- 💾 Сохранять свои проекты в интернете
- 📝 Отслеживать историю изменений
- 🤝 Делиться кодом с другими
- 🔐 Бесплатно хранить проекты

Это как Google Drive, но специально для программистов!

## Установка Git

Git - это программа, которая позволяет работать с GitHub.

### Для Windows:
1. Скачай с https://git-scm.com/download/win
2. Запусти установщик
3. Нажимай "Next" во всех диалогах

### Для macOS:
```bash
brew install git
```

### Для Linux (Ubuntu/Debian):
```bash
sudo apt-get install git
```

### Проверка что Git установлен:
```bash
git --version
```

## Первые команды

### Шаг 1: Настройка Git

Сначала скажи Git свое имя и почту:

```bash
git config --global user.name "Твое Имя"
git config --global user.email "твоя.почта@mail.com"
```

### Шаг 2: Клонирование репозитория (первый раз)

Это нужно сделать один раз. Это скачает все файлы на твой компьютер:

```bash
# Скопируй эту ссылку из GitHub
git clone https://github.com/твой-юзер/test.git

# Зайди в папку
cd test
```

### Шаг 3: Основные команды

После первого клонирования, используй эти команды:

```bash
# Обновить файлы с GitHub (перед работой)
git pull origin

# Посмотреть какие файлы изменились
git status

# Добавить файлы для сохранения
git add .

# Сохранить с комментарием что ты сделал
git commit -m "Добавил новый проект"

# Загрузить все в GitHub
git push origin
```

## Как загружать свои проекты

### Вариант 1: Если у тебя уже есть папка с проектом

```bash
# 1. Зайди в главную папку репозитория
cd ~/test

# 2. Создай папку для своего проекта
mkdir projects/мой-первый-проект

# 3. Скопируй свои файлы в эту папку
cp -r /путь/к/твоему/проекту/* projects/мой-первый-проект/

# 4. Если у тебя нет requirements.txt, создай его:
pip freeze > projects/мой-первый-проект/requirements.txt

# 5. Добавь файлы
git add projects/мой-первый-проект/

# 6. Сохрани с комментарием
git commit -m "Добавил проект: мой-первый-проект"

# 7. Загрузи в GitHub
git push origin
```

### Вариант 2: Создание проекта с нуля

```bash
# 1. Перейди в папку projects
cd ~/test/projects

# 2. Скопируй шаблон
cp -r ../template мой-проект
cd мой-проект

# 3. Отредактируй файлы
# - main.py - твой код
# - requirements.txt - зависимости
# - README.md - описание

# 4. Загрузи как в Варианте 1
```

## Частые вопросы

### ❓ Что такое "commit"?
Это снимок твоего кода на определенный момент времени. Как сохранение игры! 💾

### ❓ Что такое "push"?
Это загрузка твоих изменений на GitHub.

### ❓ Что такое "pull"?
Это скачивание последних изменений с GitHub на твой компьютер.

### ❓ Что делать если я сделал ошибку в commit?
```bash
# Посмотри историю
git log

# Откати последний commit (осторожно!)
git reset --soft HEAD~1
```

### ❓ Как посмотреть мои файлы на GitHub?
Просто открой https://github.com/твой-юзер/test

### ❓ Можно ли удалить проект?
Да! Удали папку и загрузи:
```bash
rm -rf projects/старый-проект
git add -A
git commit -m "Удалил старый проект"
git push origin
```

### ❓ Зачем нужен requirements.txt?
Это файл со списком библиотек. Когда кто-то скачает твой проект, он может установить все зависимости командой:
```bash
pip install -r requirements.txt
```
